## 1. `EmbeddingProvider` 配置与注入问题
- 文件：src/main/java/com/example/httpreading/memory/embedding/provider/EmbeddingProvider.java
- 问题概述：类使用 `@Value` 注入 `model.embedding.*` 配置，但代码中存在静态访问和 `new EmbeddingProvider()` 的情形；通过 `new` 创建的实例不会触发 Spring 注入，导致字段为 null。
- 可能后果：`apiKey` / `baseUrl` 为空，构造 `DashScopeEmbedding(modelName, apiKey, baseUrl)` 时出现请求失败或 NPE。

## 2. `application.yml` 的重复/混淆键
- 文件：src/main/resources/application.yml
- 问题概述：同时存在顶层 `model.apiKey` 与 `model.embedding.apiKey`，可能引起混淆，不明确哪个 key 为运行时实际使用。
- 建议：统一使用 `model.embedding.*` 或清晰注释用途。

## 3. `baseUrl` 的占位默认值风险
- 文件：src/main/java/com/example/httpreading/memory/embedding/provider/EmbeddingProvider.java
- 问题概述：缺少 `baseUrl` 时代码会使用占位字符串 `https://api.dashscope.example`，但该地址非真实服务，导致隐性失败。
- 建议：部署阶段强制校验必需配置或抛出明确错误，避免使用虚假占位值。

## 4. 环境变量回退不一致
- 文件：src/main/java/com/example/httpreading/memory/embedding/provider/EmbeddingProvider.java
- 问题概述：部分配置项保留了 `System.getenv(...)` 回退（如 `EMBED_MODEL_NAME`），但 `apiKey`/`baseUrl` 可能已经移除回退逻辑，部署只用 env 的场景会失败。
- 建议：明确优先级（config > env 或 env > config）并实现一致回退。

## 5. `WorkingMemory.remove()` 的并发修改风险
- 文件：src/main/java/com/example/httpreading/memory/types/WorkingMemory.java
- 问题概述：在 `for-each` 遍历 `memories` 时直接调用 `memories.remove(item)` 会抛出 `ConcurrentModificationException`。
- 建议：使用 `Iterator.remove()` 或先记录要删除的元素，循环结束后再删除。
